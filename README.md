# مساعد مَرجِع الذكي ⚖️

> مساعد قانوني ذكي يساعد المستخدمين في فهم القضايا القانونية والمواد النظامية المرتبطة بها، مبني باستخدام Python وStreamlit، مع دعم كامل للغة العربية وتقنيتَي التعرف على الكلام (ASR) وتحويل النص إلى صوت (TTS).

---

## 📋 وصف المشروع

مساعد مَرجِع الذكي هو تطبيق ويب تفاعلي يعمل كمساعد قانوني ذكي للبحث في الاستفسارات القانونية والمواد النظامية. يمكن للمستخدم كتابة استفساره أو التحدث مباشرة عبر الميكروفون، ويتلقى ردوداً قانونية منظمة مع المواد النظامية ذات الصلة.

---

## ✨ المميزات الرئيسية

- 🎤 **التعرف على الكلام (ASR):** تسجيل الصوت مباشرة وتحويله إلى نص عربي
- 🔊 **تحويل النص إلى صوت (TTS):** قراءة الردود صوتياً بشكل طبيعي
- 🔍 **كشف القضايا القانونية:** تحديد نوع القضية تلقائياً من وصف المستخدم
- 📖 **عرض المواد النظامية:** استعراض المواد القانونية المرتبطة مع شرح تفصيلي
- 🌐 **واجهة عربية كاملة:** تصميم RTL احترافي بألوان مستوحاة من نظام ناجز
- 💬 **محادثة متعددة الأدوار:** تدفق محادثة منطقي من وصف القضية حتى الخلاصة القانونية
- 🧹 **تنظيف ذكي للنص:** معالجة النص قبل إرساله لـ TTS لضمان قراءة طبيعية

## 🛠️ التقنيات المستخدمة

| التقنية | الاستخدام |
|---|---|
| `Python 3` | لغة البرمجة الرئيسية |
| `Streamlit` | إطار عمل واجهة المستخدم |
| `OpenAI SDK` | التواصل مع نماذج ASR و TTS |
| `elm-asr` | نموذج التعرف على الكلام العربي |
| `elm-tts` | نموذج تحويل النص إلى صوت عربي |
| `difflib` | المقارنة النصية الضبابية |
| `python-dotenv` | إدارة متغيرات البيئة |
| `re` | معالجة النصوص العربية بالتعبيرات النمطية |

## ⚙️ متغيرات البيئة المطلوبة

أنشئ ملف `.env` في مجلد المشروع وأضف:

```env
OPENAI_API_KEY=your_api_key_here
```

> **ملاحظة:** يستخدم المشروع `base_url` مخصص لنماذج اللغة العربية. تأكد من أن المفتاح صالح للـ endpoint المستخدم.

## 🚀 طريقة تشغيل المشروع

### 1. استنساخ المستودع

```bash
git clone https://github.com/bbusshhrraa/judge-assistant.git
cd judge-assistant
```

### 2. إنشاء بيئة افتراضية

```bash
python3 -m venv .venv
source .venv/bin/activate   # macOS/Linux
# أو على Windows:
.venv\Scripts\activate
```

### 3. تثبيت المكتبات

```bash
pip install -r requirements.txt
```

### 4. إضافة مفتاح API

```bash
echo "OPENAI_API_KEY=your_api_key_here" > .env
```

### 5. تشغيل التطبيق

```bash
streamlit run app.py
```

افتح المتصفح على: `http://localhost:8501`

## 🎙️ ملاحظات حول ASR و TTS

- **ASR (التعرف على الكلام):** يستخدم نموذج `elm-asr` عبر واجهة `st.audio_input` المدمجة في Streamlit. يدعم اللغة العربية فقط (`language="ar"`). الصوت المسجل يُحفظ كـ `speech.wav` ثم يُرسل للنموذج.

- **TTS (تحويل النص إلى صوت):** يستخدم نموذج `elm-tts`. يمر النص بمرحلة تنظيف تزيل الرموز والتنسيق غير المناسب للقراءة الصوتية قبل الإرسال للنموذج.

- **نقطة النهاية:** يعتمد المشروع على `https://elmodels.ngrok.app/v1` كـ base URL للنماذج. هذا العنوان قد يتغير — تأكد من تحديثه في `app.py` عند الحاجة.

## 🔐 الأمان

- ملف `.env` غير مرفوع لـ GitHub لحماية مفتاح API
- لا تشارك مفتاح API مع أي أحد
- عند النشر على Streamlit Cloud، أضف المفتاح عبر **Secrets** في لوحة التحكم
